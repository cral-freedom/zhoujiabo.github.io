二维数组与二维指针

2020-04-10
  • 算法类

  1. 指针包含两个方面：地址值，和所指向的数据类型

  2. 解引用操作符，会根据指针当前的地址值，以及所指向的数据类型，访问一块连续的内存空间（大小由指针所指向的数据类型决定），将这块空间的内容转换成相应的数据类型，并返回左值

  3. 因此，两个指针的值相同（指向相同的地址)，但数据类型不同，解引用取到的值也是不同的。

    验证大小端：
    int num = 0x12345678;
    char pc = #
    printf(“&num:%x\tpc:%x\n”,&num,pc);
    printf(“%x:%x(H)\t”,pc,    pc);
    打印得到：
    &num:61fe14 pc:61fe14
    61fe14:78(H)
    由上面结果可知，低地址存放低位，为小端模式。

  4. 对数组名的认识：作为右值时数组名可视为指针常量；作为左值时，例如sizeof，则不能视为指针。//sizeof(数组名) 返回的是数组大小*每个元素占字节数；而sizeof(一个指针)返回4或8。

  5. 数组和指针参数是如何被编译器修改的？

     数组名被改写成一个指针参数这个规则不是递归定义的，也就是说虽然我们可以将一维数组名传递给一维指针，但不能将二维数组传递给二维指针，如上图，二维数组匹配的形参是数组指针。

    int a[2][3];
    int (p1)[3]=a //( a = &a[0]）;
    int    p2=a[0]=&a[0][0]=*a;

将二维数组名作为参数传递，形参的形式有：
void func( int a[2][3] );
void func( int a[][3] );
void func( int (*a)[2] );

参数不能为int a[][]（会报错），至少需要告诉编译器第二维的大小。

如何实现不告诉编译器数组的维数，也能进行矩阵（多维数组）运算呢？

答案就是，模仿编译器的行为把arr[i][j]进行手工转变。
